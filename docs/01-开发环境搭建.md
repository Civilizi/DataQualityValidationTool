# 开发环境搭建

本文档介绍数据质量校验工具（Data Quality Validation Tool）的开发环境搭建步骤。

## 环境要求

| 依赖 | 最低版本 | 说明 |
|------|----------|------|
| Node.js | >= 20.0 | 使用 `node --version` 检查 |
| npm | >= 10.0 | 或使用 pnpm >= 9.0 |
| Git | 任意版本 | 版本控制 |
| DashScope API Key | - | 阿里百炼平台申请，用于 AI 解析与诊断 |

## 项目初始化步骤

### 1. 创建 Next.js 项目

```bash
npx create-next-app@latest data-quality-tool \
  --typescript --tailwind --app --src-dir --import-alias "@/*"
```

### 2. 安装核心依赖

```bash
# UI 组件库
npm install antd @ant-design/icons

# Excel 处理
npm install xlsx

# 数据库（SQLite + Drizzle ORM）
npm install drizzle-orm better-sqlite3

# AI 服务（DashScope 兼容 OpenAI 接口）
npm install openai

# 前端工具
npm install spark-md5       # 前端 MD5 计算（分块上传校验）
npm install uuid            # 生成唯一 ID

# 开发依赖
npm install -D drizzle-kit  # 数据库迁移工具
npm install -D @types/spark-md5
npm install -D @types/uuid
```

### 3. 初始化 Drizzle ORM

```bash
npx drizzle-kit init
```

### 4. 设置环境变量

```bash
cp .env.example .env.local
```

## 环境变量 (.env.example)

```env
# DashScope AI 配置
DASHSCOPE_API_KEY=sk-xxx
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
DASHSCOPE_MODEL=qwen-plus

# 数据库路径
DATABASE_PATH=./data/dqvt.db

# 文件存储目录
UPLOAD_DIR=./uploads

# 分块上传配置（单位：字节，默认 5MB）
CHUNK_SIZE=5242880
```

> **注意**：`.env.local` 不应提交到版本控制，已在 `.gitignore` 中排除。

## package.json scripts

```json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "db:generate": "drizzle-kit generate",
    "db:migrate": "drizzle-kit migrate",
    "db:seed": "tsx src/lib/db/seed.ts",
    "db:studio": "drizzle-kit studio"
  }
}
```

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动开发服务器，默认端口 3000 |
| `npm run build` | 生产环境构建 |
| `npm run start` | 启动生产服务器 |
| `npm run db:generate` | 根据 schema 生成迁移文件 |
| `npm run db:migrate` | 执行迁移，创建/更新数据库表 |
| `npm run db:seed` | 填充初始数据（业务域、默认规则等） |
| `npm run db:studio` | 启动 Drizzle Studio（Web 数据库管理界面） |

## 开发启动流程

```bash
# 1. 生成迁移文件
npm run db:generate

# 2. 执行迁移，创建 SQLite 数据库
npm run db:migrate

# 3. 启动开发服务器
npm run dev
```

启动成功后，访问 **http://localhost:3000** 即可使用。

首次使用建议先运行 `npm run db:seed` 填充基础数据。

## 常见问题排查

### better-sqlite3 编译失败

如果安装 `better-sqlite3` 时出现原生模块编译错误：

```bash
npm install --build-from-source better-sqlite3
```

确保系统已安装 C++ 编译工具：
- **Windows**：安装 [Build Tools for Visual Studio](https://visualstudio.microsoft.com/visual-cpp-build-tools/)
- **macOS**：运行 `xcode-select --install`
- **Linux**：安装 `build-essential` 和 `python3`

### SQLite 数据库文件不存在

确保 `data/` 目录存在。首次执行 `npm run db:migrate` 时会自动创建该目录和数据库文件。如果手动删除了数据库，需重新运行迁移。

### API Key 未配置

启动后访问系统设置页面（`/settings`）配置 DashScope API Key，或直接写入 `.env.local` 中的 `DASHSCOPE_API_KEY` 变量。未配置 API Key 时 AI 功能不可用，但不影响核心校验流程。

### 端口 3000 被占用

```bash
# 使用其他端口启动
PORT=3001 npm run dev
```

### 迁移文件与数据库不同步

```bash
# 删除旧迁移文件重新生成
rm -rf drizzle/
npm run db:generate
npm run db:migrate
```

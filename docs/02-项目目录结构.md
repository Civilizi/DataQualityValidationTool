## 目录详解

### `src/app/` — Next.js App Router 路由

**职责**：定义应用的页面路由结构，是 Next.js 15 的核心路由层。采用 App Router 模式，每个目录对应一个路由路径。

**关键文件**：
- `layout.tsx` — 根布局，负责注入 Ant Design 的 `ConfigProvider` 主题配置和全局 Provider
- `page.tsx` — 首页入口，通常重定向到 `/dashboard`
- `globals.css` — 全局 CSS，包含 Tailwind 指令和自定义样式变量

**与 `app/api/` 的关系**：页面组件通过 `fetch` 调用 `app/api/` 下的 API 路由获取数据，形成前后端分离的 BFF（Backend for Frontend）架构。

---

### `src/app/(main)/` — 主功能路由组

**职责**：承载所有业务页面，通过括号 `(main)` 创建路由组，不增加 URL 层级。共享同一套侧边栏布局。

**关键文件**：
- `layout.tsx` — 主布局组件，包含侧边栏导航、顶部业务域选择器和主内容区

**各子路由**：

| 路由 | 页面 | 功能 |
|------|------|------|
| `dashboard/` | 工作台 | 全局概览：统计卡片、近期任务、通过率趋势 |
| `standards/` | 数据标准 | 标准列表展示、上传入口、AI 解析规则确认 |
| `validation/` | 数据校验 | 创建校验任务、字段匹配、进度追踪、结果导出 |
| `assets/` | 素材池 | 素材上传、版本管理、分块上传支持 |
| `tasks/` | 任务中心 | 校验任务管理、进度监控、任务详情 |
| `history/` | 历史记录 | 操作日志、版本追溯、变更对比 |
| `settings/` | 系统设置 | AI 配置、Prompt 模板管理、别名映射 |

**与 `components/` 的关系**：每个路由页面引入对应子目录的组件，实现页面组装。例如 `standards/page.tsx` 使用 `components/standards/` 下的组件。

---

### `src/app/api/` — API Routes（后端逻辑）

**职责**：提供 RESTful API 端点，处理所有后端业务逻辑。Next.js 在同一个进程中运行，无需独立后端服务。

**关键目录**：

| 目录 | 职责 |
|------|------|
| `domains/` | 业务域的增删改查，支持多租户隔离 |
| `standards/` | 数据标准文件上传、AI 解析、规则提取与管理 |
| `rules/` | 规则库的 CRUD，定义校验规则模板 |
| `assets/` | 素材文件上传（含分块上传、MD5 校验、分片合并） |
| `validation/` | 校验任务创建、执行调度、结果查询与导出 |
| `history/` | 历史操作记录的写入与查询 |
| `settings/` | AI 参数配置、Prompt 模板 CRUD、别名映射管理 |

**与 `lib/` 的关系**：API Route 调用 `lib/` 中的业务逻辑（数据库操作、规则引擎、AI 服务等），自身只负责请求解析、参数校验和响应格式化。

---

### `src/components/` — 可复用 UI 组件

**职责**：封装 Ant Design 组件和业务 UI 逻辑，提供一致的交互体验。按功能域组织，与路由结构一一对应。

**关键目录**：

| 目录 | 职责 |
|------|------|
| `layout/` | 全局布局组件：侧边栏、顶部导航、主内容容器 |
| `dashboard/` | 工作台组件：统计卡片、任务概览表格、趋势图表 |
| `standards/` | 数据标准组件：文件上传器、规则预览表格、AI 解析确认弹窗 |
| `validation/` | 校验组件：任务创建表单、字段匹配表、进度条、结果表格 |
| `assets/` | 素材组件：分块上传器、素材列表、版本历史列表 |
| `tasks/` | 任务组件：任务管理表格、状态标签、详情抽屉面板 |
| `history/` | 历史组件：时间筛选器、操作日志表格、版本追溯视图 |
| `settings/` | 设置组件：AI 配置表单、Prompt 编辑器、别名映射表格 |
| `common/` | 通用组件：文件选择器、确认对话框、空状态提示、错误边界 |

**设计原则**：组件按功能域隔离，避免跨域依赖。`common/` 中的组件可被任意模块引用。

---

### `src/lib/` — 核心逻辑库

**职责**：承载所有非 UI 业务逻辑，是整个应用的核心。包含数据库、规则引擎、AI 服务、文件处理等模块。

#### `lib/db/` — 数据库层

**职责**：封装 Drizzle ORM 的数据库操作。

| 文件 | 说明 |
|------|------|
| `schema.ts` | 13 张表的 Drizzle 表定义（业务域、标准、规则、素材、任务、结果等） |
| `relations.ts` | 表之间的关联关系（外键、一对多、多对多） |
| `index.ts` | 数据库连接实例导出，供全局引用 |

#### `lib/engine/` — 规则引擎

**职责**：执行数据校验的核心引擎，按规则类型对素材数据进行逐条校验。

| 文件 | 说明 |
|------|------|
| `index.ts` | 引擎入口，接收任务和素材数据，调度执行 |
| `executor.ts` | 校验执行器，遍历数据行并调用对应规则 |
| `batch-runner.ts` | 分批执行控制器，控制内存占用，支持大文件 |
| `checkpoint.ts` | Checkpoint 管理，支持断点续跑 |
| `rules/` | 规则实现子目录 |

**规则实现（`lib/engine/rules/`）**：
- `base.ts` — 规则抽象基类，定义统一接口 `validate()` 方法
- `not-null.ts` — 非空校验
- `regex.ts` — 正则表达式格式校验
- `length-range.ts` — 字符串长度范围校验
- `enum-check.ts` — 枚举值校验
- `unique-check.ts` — 字段唯一性校验
- `date-format.ts` — 日期格式校验
- `value-range.ts` — 数值范围校验
- `cross-field.ts` — 跨字段逻辑校验（如 A > B）
- `cross-table.ts` — 跨表关联校验

#### `lib/ai/` — AI 服务

**职责**：封装与阿里百炼 DashScope（通义千问）的交互，提供标准解析、字段匹配、问题诊断等 AI 能力。

| 文件 | 说明 |
|------|------|
| `dashscope.ts` | DashScope API 客户端，基于 OpenAI SDK 封装 |
| `prompts/` | Prompt 模板目录 |
| `fallback.ts` | AI 服务降级策略（超时、限流、失败时的兜底逻辑） |

**Prompt 模板（`lib/ai/prompts/`）**：
- `standard-parse.ts` — 数据标准文档解析 Prompt
- `field-match.ts` — 标准字段与素材字段自动匹配 Prompt
- `issue-diagnosis.ts` — 校验结果问题诊断 Prompt
- `report-gen.ts` — 校验报告自动生成 Prompt
- `index.ts` — 统一导出入口

#### `lib/upload/` — 文件上传处理

**职责**：处理前端分块上传的后端逻辑。

| 文件 | 说明 |
|------|------|
| `chunk-handler.ts` | 接收和存储上传分块 |
| `md5-calculator.ts` | 文件 MD5 计算，用于去重和完整性校验 |
| `merge-handler.ts` | 分片合并，将临时分块拼接为完整文件 |

#### `lib/excel/` — Excel 处理

**职责**：Excel 文件的解析和生成。

| 文件 | 说明 |
|------|------|
| `parser.ts` | 解析 Excel/CSV 文件，提取表头和数据行 |
| `generator.ts` | 生成带样式的 Excel 报告（校验结果导出） |

#### `lib/versioning/` — 版本管理

**职责**：管理素材和标准的版本控制。

| 文件 | 说明 |
|------|------|
| `version-manager.ts` | 版本号自动生成（语义化版本），业务域 + 标准 + 素材 + 版本四元绑定 |

#### `lib/utils/` — 工具函数

**职责**：全局通用的辅助函数。

| 文件 | 说明 |
|------|------|
| `constants.ts` | 全局常量：批次大小、默认颜色、状态枚举等 |
| `formatters.ts` | 通用格式化：数字、日期、文件大小、百分比等 |

---

### `src/hooks/` — React Hooks

**职责**：封装可复用的 React 状态逻辑，供组件调用。

| 文件 | 说明 |
|------|------|
| `useDomain.ts` | 当前业务域的读取、切换、持久化状态管理 |
| `useUpload.ts` | 分块上传进度跟踪、取消、重试逻辑 |
| `useValidationProgress.ts` | 校验任务进度轮询，定时查询执行状态并更新 UI |

---

### `src/types/` — TypeScript 类型定义

**职责**：定义全局 TypeScript 接口和类型，确保前后端数据结构一致。

| 文件 | 说明 |
|------|------|
| `domain.ts` | 业务域相关类型 |
| `standard.ts` | 数据标准、规则定义类型 |
| `rule.ts` | 规则引擎输入输出类型 |
| `asset.ts` | 素材文件、版本信息类型 |
| `validation.ts` | 校验任务、执行结果、校验记录类型 |
| `ai.ts` | AI 服务请求/响应、Prompt 模板类型 |
| `history.ts` | 历史记录、操作日志类型 |

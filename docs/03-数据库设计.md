# 数据库设计

**版本**：V1.0
**编制日期**：2026年5月28日

本文档描述数据质量校验工具的 SQLite 数据库架构，使用 Drizzle ORM 定义。数据库按业务域组织，覆盖标准管理、规则库、素材池、校验任务、执行日志、结果存储、交付成果、AI 配置、审计日志等全部核心实体。

## 二、表结构详细定义

### 2.1 business_domains — 业务域

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PRIMARY KEY | 业务域唯一标识，如 `hr`、`finance`、`sales` |
| name | TEXT | UNIQUE NOT NULL | 业务域名称 |
| description | TEXT | | 业务域描述 |
| status | TEXT | NOT NULL, DEFAULT 'active' | 状态：`active` / `inactive` |
| created_at | TEXT | NOT NULL | 创建时间 ISO 8601 |
| updated_at | TEXT | NOT NULL | 更新时间 ISO 8601 |

### 2.2 data_standards — 数据标准

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PRIMARY KEY | 标准唯一标识 |
| domain_id | TEXT | NOT NULL, FK→business_domains(id) | 所属业务域 |
| name | TEXT | NOT NULL | 标准原始文件名（不含路径） |
| display_name | TEXT | NOT NULL | 显示名称，含版本号如 `HR标准_v1` |
| version | INTEGER | NOT NULL, DEFAULT 1 | 版本号，同名递增 |
| file_path | TEXT | NOT NULL | 文件存储路径 |
| file_size | INTEGER | NOT NULL | 文件大小（字节） |
| file_hash | TEXT | NOT NULL | 文件 MD5 哈希 |
| parse_status | TEXT | NOT NULL, DEFAULT 'pending' | `pending` / `parsing` / `parsed` / `confirmed` / `rejected` |
| total_rules | INTEGER | DEFAULT 0 | AI 解析出的规则总数 |
| confirmed_rules | INTEGER | DEFAULT 0 | 已确认的规则数 |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

**索引**：
- `idx_standards_domain_id` ON `domain_id`
- `idx_standards_file_hash` ON `file_hash`

### 2.3 validation_rules — 校验规则库

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PRIMARY KEY | 规则唯一标识 |
| standard_id | TEXT | NOT NULL, FK→data_standards(id) | 所属标准 |
| table_name | TEXT | NOT NULL | 表单/表名称 |
| field_name | TEXT | NOT NULL | 字段名 |
| dimension | TEXT | NOT NULL | 维度：`完整性` / `准确性` / `有效性` / `唯一性` / `一致性` |
| level | TEXT | NOT NULL | 级别：`field` / `record` / `cross_dataset` |
| original_text | TEXT | NOT NULL | 业务标准原始文本 |
| executable_type | TEXT | NOT NULL | 可执行类型，见下方枚举 |
| executable_params | TEXT (JSON) | | 执行参数，JSON 字符串 |
| severity | TEXT | NOT NULL, DEFAULT 'error' | `error` / `warning` / `info` |
| confidence | TEXT | NOT NULL, DEFAULT 'medium' | `high` / `medium` / `low` |
| status | TEXT | NOT NULL, DEFAULT 'pending' | `pending` / `confirmed` / `rejected` / `modified` |
| sort_order | INTEGER | DEFAULT 0 | 排序序号 |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

**executable_type 枚举值**：
`not_null` / `regex` / `length_range` / `enum_check` / `unique` / `date_format` / `value_range` / `cross_field` / `cross_table`

**索引**：
- `idx_rules_standard_id` ON `standard_id`
- `idx_rules_level` ON `level`
- `idx_rules_status` ON `status`
- `idx_rules_table_field` ON `(table_name, field_name)`

| row_count | INTEGER | DEFAULT 0 | 总行数（所有 Sheet 合计） |
| column_names | TEXT (JSON) | | 各 Sheet 列名映射，JSON 对象 |
| upload_status | TEXT | NOT NULL, DEFAULT 'uploading' | `uploading` / `uploaded` / `processing` / `ready` |

## 四、Drizzle ORM 定义示例

### 4.1 business_domains

```typescript
// src/lib/db/schema/domains.ts
import { sqliteTable, text, integer } from "drizzle-orm/sqlite-core";
import { sql } from "drizzle-orm";

export const businessDomains = sqliteTable("business_domains", {
  id: text("id").primaryKey(),
  name: text("name").notNull().unique(),
  description: text("description"),
  status: text("status", { enum: ["active", "inactive"] })
    .notNull()
    .default("active"),
  createdAt: text("created_at")
    .notNull()
    .default(sql`(datetime('now'))`),
  updatedAt: text("updated_at")
    .notNull()
    .default(sql`(datetime('now'))`),
});
```

### 4.2 validation_rules

```typescript
// src/lib/db/schema/rules.ts
import { sqliteTable, text, integer } from "drizzle-orm/sqlite-core";
import { sql } from "drizzle-orm";
import { dataStandards } from "./standards";
import { relations, index } from "drizzle-orm";

export const validationRules = sqliteTable(
  "validation_rules",
  {
    id: text("id").primaryKey(),
    standardId: text("standard_id")
      .notNull()
      .references(() => dataStandards.id, { onDelete: "cascade" }),
    tableName: text("table_name").notNull(),
    fieldName: text("field_name").notNull(),
    dimension: text("dimension", {
      enum: ["完整性", "准确性", "有效性", "唯一性", "一致性"],
    }).notNull(),
    level: text("level", {
      enum: ["field", "record", "cross_dataset"],
    }).notNull(),
    originalText: text("original_text").notNull(),
    executableType: text("executable_type", {
      enum: [
        "not_null", "regex", "length_range", "enum_check",
```

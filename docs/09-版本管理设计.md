## 9.1 四元版本绑定机制

每次校验任务执行时，系统绑定四个版本号，确保校验结果可复现、可溯源、可对比。这是本工具的核心设计之一。

```typescript
interface VersionBinding {
  // 1. 标准版本
  standardId: string;       // data_standards.id
  standardVersion: number;  // 如 3，表示该标准的第 3 个版本

  // 2. 素材版本
  assetIds: string[];       // data_assets.id 列表
  assetVersions: number[];  // 对应版本号列表，与 assetIds 一一对应

  // 3. 任务版本
  taskId: string;           // validation_tasks.id
  taskVersion: number;      // 初始为 1，同一任务重跑递增

  // 4. 成果版本
  deliverableVersion: number; // 初始为 1，同一任务重复导出递增
}
```

### 9.1.1 为什么需要四元绑定

| 维度 | 不绑定的问题 | 绑定后的效果 |
|------|-------------|-------------|
| 标准版本 | 无法确认某次校验使用的是哪个版本的标准 | 可复现：同样标准版本+同样数据 = 同样结果 |
| 素材版本 | 数据更新后无法追溯历史校验用的是哪份数据 | 可溯源：任意历史结果可回溯到原始数据 |
| 任务版本 | 同一任务重跑后无法区分新旧结果 | 可对比：v1 结果和 v2 结果可以并排比较 |
| 成果版本 | 同一任务多次导出文件覆盖 | 可追溯：每次导出都有记录 |

## 9.2 版本号生成规则

| 实体 | 版本规则 | 示例 | 触发时机 |
|------|---------|------|---------|
| 数据标准 | 同名文件重复上传 → `version = max(已有版本) + 1` | `标准_v1` → `标准_v2` → `标准_v3` | 上传标准文件 |
| 素材资产 | 同名文件重复上传 → `version = max(已有版本) + 1`；MD5 相同则秒传 | `数据_v1` → `数据_v2` | 上传数据文件 |
| 校验任务 | 首次创建 → `taskVersion = 1`；同一任务重跑 → `taskVersion++` | `task_v1` → `task_v2` | 创建/重跑任务 |
| 交付成果 | 首次导出 → `deliverableVersion = 1`；同一任务重新导出 → `version++` | `deliverable_v1` → `deliverable_v2` | 导出成果文件 |

### 9.2.1 秒传逻辑

当上传文件的 MD5 哈希与已有资产完全一致时，触发秒传：

```
用户上传文件 "员工数据.xlsx"
    ↓
系统计算 MD5: abc123...
    ↓
查询 data_assets 表: WHERE file_hash = 'abc123...' AND domain_id = ?
    ↓
已存在？──是──→ 不创建新文件，返回已有资产
  │                  不增加版本号（因为是同一份数据）
  │                  但可以在 asset_usage 表记录此次引用
 否
  ↓
创建新 data_assets 记录
  version = max(同名已有版本) + 1
```

## 9.3 版本号生成器

```typescript
// lib/versioning/version-manager.ts

import { db } from '../db';
import { dataStandards, dataAssets, validationTasks } from '../db/schema';
import { eq, and, max } from 'drizzle-orm';

class VersionManager {

  /**
   * 获取下一个版本号
   * @param tableName 表名标识（'standards' | 'assets'）
   * @param name 文件/资产名称
   * @param domainId 业务域 ID
   */
  async getNextVersion(
    tableName: 'standards' | 'assets',
    name: string,
    domainId: string
  ): Promise<number> {
    const table = tableName === 'standards' ? dataStandards : dataAssets;

    const existing = await db.select({ version: table.version })
      .from(table)
      .where(and(
        eq(table.name, name),
        eq(table.domainId, domainId),
      ));

    if (existing.length === 0) return 1;

    const maxVersion = Math.max(...existing.map(e => e.version));
    return maxVersion + 1;
  }

  /**
   * 生成显示名称
   * @param originalName 原始文件名（不含扩展名）
   * @param version 版本号
   */
  generateDisplayName(originalName: string, version: number): string {
    // 移除扩展名
    const baseName = originalName.replace(/\.(xlsx?|csv)$/i, '');
    return `${baseName}_v${version}`;
  }

  /**
   * 创建版本绑定
   * 在创建校验任务时调用
   */
  async createBinding(
    taskId: string,
    standardId: string,
    assetIds: string[]
  ): Promise<VersionBinding> {
    // 查询标准版本
    const standard = await db.query.dataStandards.findFirst({
      where: eq(dataStandards.id, standardId),
    });
    if (!standard) {
      throw new Error(`标准不存在：${standardId}`);
    }

    // 查询素材版本
    const assets = await db.query.dataAssets.findMany({
      where: inArray(dataAssets.id, assetIds),
    });
    if (assets.length !== assetIds.length) {
      throw new Error('部分素材不存在');
    }

    const binding: VersionBinding = {
      standardId,
      standardVersion: standard.version,
      assetIds,
      assetVersions: assets.map(a => a.version),
      taskId,
      taskVersion: 1, // 新建任务初始版本
      deliverableVersion: 1, // 初始成果版本
    };

    // 保存绑定到任务记录
    await db.insert(validationTasks).values({
      id: taskId,
      standardId,
      standardVersion: standard.version,
      taskVersion: 1,
      deliverableVersion: 1,
      status: 'pending',
      createdAt: new Date(),
      updatedAt: new Date(),
    });

    // 保存任务-素材关联
    for (const assetId of assetIds) {
      await db.insert(taskAssetRelations).values({
        taskId,
        assetId,
      });
    }

    return binding;
  }

  /**
   * 查询历史版本绑定
   */
  async getBinding(taskId: string): Promise<VersionBinding | null> {
    const task = await db.query.validationTasks.findFirst({
      where: eq(validationTasks.id, taskId),
    });
    if (!task) return null;

    // 查询关联素材
    const relations = await db.query.taskAssetRelations.findMany({
      where: eq(taskAssetRelations.taskId, taskId),
    });
    const assetIds = relations.map(r => r.assetId);

    // 查询素材版本
    const assets = await db.query.dataAssets.findMany({
      where: inArray(dataAssets.id, assetIds),
    });

    return {
      standardId: task.standardId,
      standardVersion: task.standardVersion,
      assetIds,
      assetVersions: assets.map(a => a.version),
      taskId: task.id,
      taskVersion: task.taskVersion,
      deliverableVersion: task.deliverableVersion,
    };
  }

  /**
   * 递增任务版本号（重跑时调用）
   */
  async incrementTaskVersion(taskId: string): Promise<number> {
    const task = await db.query.validationTasks.findFirst({
      where: eq(validationTasks.id, taskId),
    });
    if (!task) throw new Error(`任务不存在：${taskId}`);

    const newVersion = task.taskVersion + 1;
    await db.update(validationTasks)
      .set({ taskVersion: newVersion, updatedAt: new Date() })
      .where(eq(validationTasks.id, taskId));

    return newVersion;
  }

  /**
   * 递增成果版本号（导出时调用）
   */
  async incrementDeliverableVersion(taskId: string): Promise<number> {
    const task = await db.query.validationTasks.findFirst({
      where: eq(validationTasks.id, taskId),
    });
    if (!task) throw new Error(`任务不存在：${taskId}`);

    const newVersion = task.deliverableVersion + 1;
    await db.update(validationTasks)
      .set({ deliverableVersion: newVersion, updatedAt: new Date() })
      .where(eq(validationTasks.id, taskId));

    return newVersion;
  }
}

export const versionManager = new VersionManager();
```

## 9.4 追溯查询

通过 `validation_tasks.id` 可以追溯完整的输入输出链：

```
validation_tasks.id
│
├── → data_standards（使用的标准版本）
│   └── → validation_rules（该标准下固化的规则集）
│        └── → rule_name, rule_type, params, severity
│
├── → data_assets（使用的素材版本列表）
│   └── → file_path, file_hash, sheet_count, sheet_metadata
│
├── → execution_logs（执行过程记录）
│   └── → phase, batch_number, checkpoint, status, timestamp
│
├── → validation_results（校验结果明细）
│   └── → rule_id, sheet_name, row_index, field_name, severity, message
│        └── → ai_diagnosis, ai_suggestion（AI 辅助填充）
│
└── → deliverables（导出的成果文件）
     └── → file_path, file_type, version, created_at
```

    },
  });

  return task;
}

      standard: true,
      assets: true,
    },
  });
}

## 9.5 版本对比

### 9.5.1 标准版本对比

对比同一标准 v1 和 v2 的规则差异，识别新增、修改、删除的规则。

```typescript
// lib/versioning/diff.ts

interface RuleDiff {
  ruleId: string;
  tableName: string;
  fieldName: string;
  changeType: 'added' | 'modified' | 'deleted';
  oldRule?: ValidationRule;
  newRule?: ValidationRule;
  diffSummary: string;  // 人类可读的差异描述
}

export async function diffStandardVersions(
  standardId: string,
  versionA: number,
  versionB: number
): Promise<RuleDiff[]> {
  // 查询两个版本的规则
  const rulesA = await db.query.validationRules.findMany({
    where: and(
      eq(validationRules.standardId, standardId),
      eq(validationRules.standardVersion, versionA),
    ),
  });

  const rulesB = await db.query.validationRules.findMany({
    where: and(
      eq(validationRules.standardId, standardId),
      eq(validationRules.standardVersion, versionB),
    ),
  });

  const diffs: RuleDiff[] = [];

  // 构建映射：ruleId → rule
  const mapA = new Map(rulesA.map(r => [generateRuleKey(r), r]));
  const mapB = new Map(rulesB.map(r => [generateRuleKey(r), r]));

  // 新增的规则（B 有 A 没有）
  for (const [key, ruleB] of mapB) {
    if (!mapA.has(key)) {
      diffs.push({
        ruleId: ruleB.id,
        tableName: ruleB.tableName,
        fieldName: ruleB.fieldName,
        changeType: 'added',
        newRule: ruleB,
        diffSummary: `新增规则：${ruleB.originalText}`,
      });
    }
  }

  // 删除的规则（A 有 B 没有）
  for (const [key, ruleA] of mapA) {
    if (!mapB.has(key)) {
      diffs.push({
        ruleId: ruleA.id,
        tableName: ruleA.tableName,
        fieldName: ruleA.fieldName,
        changeType: 'deleted',
        oldRule: ruleA,
        diffSummary: `删除规则：${ruleA.originalText}`,
      });
    }
  }

  // 修改的规则（两边都有但内容不同）
  for (const [key, ruleA] of mapA) {
    const ruleB = mapB.get(key);
    if (ruleB) {
      const changes = compareRules(ruleA, ruleB);
      if (changes.length > 0) {
        diffs.push({
          ruleId: ruleB.id,
          tableName: ruleB.tableName,
          fieldName: ruleB.fieldName,
          changeType: 'modified',
          oldRule: ruleA,
          newRule: ruleB,
          diffSummary: `修改规则：${changes.join('；')}`,
        });
      }
    }
  }

  return diffs;
}

function generateRuleKey(rule: ValidationRule): string {
  return `${rule.tableName}::${rule.fieldName}::${rule.executableType}`;
}

function compareRules(oldRule: ValidationRule, newRule: ValidationRule): string[] {
  const changes: string[] = [];

  if (oldRule.executableType !== newRule.executableType) {
    changes.push(`规则类型从 ${oldRule.executableType} 改为 ${newRule.executableType}`);
  }

  if (JSON.stringify(oldRule.executableParams) !== JSON.stringify(newRule.executableParams)) {
    changes.push('参数已变更');
  }

  if (oldRule.severity !== newRule.severity) {
    changes.push(`严重等级从 ${oldRule.severity} 改为 ${newRule.severity}`);
  }

  if (oldRule.originalText !== newRule.originalText) {
    changes.push('规则描述已更新');
  }

  return changes;
}
```

### 9.5.2 校验结果对比

对比同一批数据在两个不同标准版本下的校验结果差异。

```typescript
interface ResultDiff {
  sheetName: string;
  rowIndex: number;
  fieldName?: string;
  statusA: 'passed' | 'failed';  // 标准 A 下的结果
  statusB: 'passed' | 'failed';  // 标准 B 下的结果
  ruleIdA?: string;
  ruleIdB?: string;
  changeType: 'now_passing' | 'now_failing' | 'rule_changed';
  description: string;
}

export async function diffValidationResults(
  taskVersionA: string,
  taskVersionB: string
): Promise<ResultDiff[]> {
  const resultsA = await db.query.validationResults.findMany({
    where: eq(validationResults.taskVersion, taskVersionA),
  });

  const resultsB = await db.query.validationResults.findMany({
    where: eq(validationResults.taskVersion, taskVersionB),
  });

  const diffs: ResultDiff[] = [];

  // 构建 key → result 映射
  const keyA = new Map(resultsA.map(r => resultKey(r)));
  const keyB = new Map(resultsB.map(r => resultKey(r)));

  // 新增的问题（B 有 A 没有 → now_failing）
  for (const [key, resultB] of keyB) {
    if (!keyA.has(key) && !resultB.passed) {
      diffs.push({
        sheetName: resultB.sheetName,
        rowIndex: resultB.rowIndex,
        fieldName: resultB.fieldName,
        statusA: 'passed',
        statusB: 'failed',
        ruleIdB: resultB.ruleId,
        changeType: 'now_failing',
        description: `新增问题：${resultB.message}`,
      });
    }
  }

  // 已解决的问题（A 有 B 没有 → now_passing）
  for (const [key, resultA] of keyA) {
    if (!keyB.has(key) && !resultA.passed) {
      diffs.push({
        sheetName: resultA.sheetName,
        rowIndex: resultA.rowIndex,
        fieldName: resultA.fieldName,
        statusA: 'failed',
        statusB: 'passed',
        ruleIdA: resultA.ruleId,
        changeType: 'now_passing',
        description: `问题已解决`,
      });
    }
  }

  // 同一位置规则变更（rule_changed）
  for (const [key, resultA] of keyA) {
    const resultB = keyB.get(key);
    if (resultB && resultA.ruleId !== resultB.ruleId && !resultA.passed && !resultB.passed) {
      diffs.push({
        sheetName: resultA.sheetName,
        rowIndex: resultA.rowIndex,
        fieldName: resultA.fieldName,
        statusA: 'failed',
        statusB: 'failed',
        ruleIdA: resultA.ruleId,
        ruleIdB: resultB.ruleId,
        changeType: 'rule_changed',
        description: `规则从 ${resultA.ruleId} 变更为 ${resultB.ruleId}`,
      });
    }
  }

  return diffs;
}

function resultKey(result: ValidationResult): string {
  return `${result.sheetName}::${result.rowIndex}::${result.fieldName || ''}`;
}
```

### 9.5.3 对比结果用途

对比结果可用于以下场景：

| 用途 | 说明 |
|------|------|
| 规则迭代效果评估 | 标准从 v1 升级到 v2，查看新增/删除/修改了多少条规则，以及这些变更对校验结果的影响 |
| 标准变更影响分析 | 修改某条规则后，之前通过的数据是否仍然通过，之前报错的数据是否已经修复 |
| 审计追踪 | 任意历史校验结果可复现、可对比，满足审计合规要求 |
| 质量趋势分析 | 同一批数据多次校验的通过率趋势，评估数据治理效果 |

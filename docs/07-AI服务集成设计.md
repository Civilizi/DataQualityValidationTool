## 7.1 技术选型

本系统通过阿里百炼 DashScope API 调用通义千问模型，实现 AI 辅助能力。DashScope 兼容 OpenAI API 格式，可直接使用 `openai` npm 包调用。

| 项目 | 选型 | 说明 |
|------|------|------|
| SDK | `openai` npm package | DashScope 兼容 OpenAI 接口，无需额外 SDK |
| Base URL | `https://dashscope.aliyuncs.com/compatible-mode/v1` | 阿里百炼兼容模式端点 |
| 默认模型 | `qwen-plus` | 推荐模型，平衡速度与质量 |
| 可选模型 | `qwen-turbo`（快速） / `qwen-max`（高精度） | 用户可在设置中切换 |
| 温度 | `0.1` | 低温度保证输出稳定性，减少随机性 |
| 响应格式 | JSON（`response_format: { type: "json_object" }`） | 结构化输出，便于程序解析 |
| 超时 | 30 秒 | 单次调用超时限制 |
| 重试 | 失败后重试 1 次 | 网络抖动容错 |

## 7.2 DashScope 服务封装

```typescript
// lib/ai/dashscope.ts

import OpenAI from 'openai';

class DashScopeService {
  private client: OpenAI | null = null;
  private config: AiConfig | null = null;

  async init(config: AiConfig) {
    this.config = config;
    this.client = new OpenAI({
      apiKey: config.apiKey,
      baseURL: config.apiBaseUrl || 'https://dashscope.aliyuncs.com/compatible-mode/v1',
    });
  }

  private ensureInitialized() {
    if (!this.client) {
      throw new Error('AI 服务未初始化，请检查 API Key 配置');
    }
  }

  async chat(messages: ChatMessage[], options?: ChatOptions): Promise<string> {
    this.ensureInitialized();

    const response = await this.client!.chat.completions.create({
      model: options?.model || this.config!.model!,
      messages,
      temperature: options?.temperature ?? 0.1,
      max_tokens: options?.maxTokens ?? 8000,
      response_format: options?.jsonMode ? { type: 'json_object' } : undefined,
    });

    return response.choices[0].message.content || '';
  }

  /**
   * 解析 JSON 响应，带重试和容错
   */
  async chatJson<T = any>(messages: ChatMessage[], options?: ChatOptions): Promise<T> {
    const raw = await this.chat(messages, { ...options, jsonMode: true });

    // 尝试解析 JSON（AI 可能在 JSON 前后加说明文字）
    try {
      return JSON.parse(raw);
    } catch {
      // 尝试提取 JSON 块
      const jsonMatch = raw.match(/\{[\s\S]*\}/);
      if (jsonMatch) {
        return JSON.parse(jsonMatch[0]);
      }
      throw new Error(`AI 返回非 JSON 格式：${raw.substring(0, 200)}`);
    }
  }

  // ========== 便捷方法 ==========

  async parseStandard(content: string, template: string): Promise<ParsedRule[]> {
    const messages: ChatMessage[] = [
      { role: 'system', content: template },
      { role: 'user', content },
    ];
    return this.chatJson<ParsedRule[]>(messages);
  }

  async matchFields(
    assetColumns: string[],
    ruleFields: string[],
    template: string
  ): Promise<FieldMatch[]> {
    const content = JSON.stringify({ assetColumns, ruleFields });
    const messages: ChatMessage[] = [
      { role: 'system', content: template },
      { role: 'user', content },
    ];
    return this.chatJson<FieldMatch[]>(messages);
  }

  async diagnoseIssue(
    issue: ValidationResult,
    template: string
  ): Promise<AIDiagnosis> {
    const content = JSON.stringify({
      sheetName: issue.sheetName,
      fieldName: issue.fieldName,
      originalValue: issue.originalValue,
      ruleMessage: issue.message,
      severity: issue.severity,
    });
    const messages: ChatMessage[] = [
      { role: 'system', content: template },
      { role: 'user', content },
    ];
    return this.chatJson<AIDiagnosis>(messages);
  }

  async generateReport(
    summary: ValidationSummary,
    template: string
  ): Promise<string> {
    const content = JSON.stringify(summary);
    const messages: ChatMessage[] = [
      { role: 'system', content: template },
      { role: 'user', content },
    ];
    return this.chat(messages);
  }
}

// 单例导出
export const dashScopeService = new DashScopeService();
```

## 7.3 四种 Prompt 模板

系统定义了四种标准 Prompt 模板，分别对应四种 AI 辅助场景。所有模板均为 TypeScript 模块导出，便于版本管理和测试。

### 7.3.1 标准解析 Prompt（standard-parse.ts）

**目的**：将自然语言的业务质量标准文件翻译为可执行的校验规则。

```typescript
// lib/ai/prompts/standard-parse.ts

export const STANDARD_PARSE_SYSTEM_PROMPT = `你是数据质量标准解析专家。你的任务是将业务方提供的自然语言数据质量标准翻译为系统可执行的校验规则。

## 输入说明
- 输入可能是不规则的 Excel 表格或文本
- 需要识别表头结构，判断哪列是表名、字段名、规则描述
- 规则描述可能包含维度标签如【完整性】【准确性】【有效性】【唯一性】【一致性】
- 规则可能涉及字段级、记录级、跨数据集级三个层次

## 输出格式
输出必须是 JSON 数组，每条规则包含以下字段：

\`\`\`json
[
  {
    "tableName": "表名",
    "fieldName": "字段名",
    "dimension": "完整性|准确性|有效性|唯一性|一致性",
    "level": "field|record|cross",
    "originalText": "原始规则文本",
    "executableType": "not_null|regex|length_range|enum_check|unique|date_format|value_range|cross_field|cross_table",
    "executableParams": {},
    "severity": "error|warning|info",
    "confidence": "high|medium|low"
  }
]
```

## executableParams 格式参照表

| executableType | executableParams 格式 | 示例 |
|---------------|---------------------|------|
| not_null | { "placeholders": ["占位符列表"] } | {"placeholders": ["无","-","待补充","NA","null"]} |
| regex | { "pattern": "正则表达式" } | {"pattern": "^\\\\d{8}$"} |
| length_range | { "min": 数字, "max": 数字 } | {"min": 2, "max": 15} |
| enum_check | { "values": ["枚举值列表"] } | {"values": ["晋升","降职","平调","离职"]} |
| unique | { "scope": "sheet" } | {"scope": "sheet"} |
| date_format | { "format": "日期格式" } | {"format": "YYYY-MM-DD"} |
| value_range | { "min": 数字, "max": 数字, "inclusive": true } | {"min": 0, "max": 150, "inclusive": true} |
| cross_field | { "condition": "条件", "thenFields": ["字段列表"], "checkType": "校验类型" } | {"condition": "异动类型 == '晋升'", "thenFields": ["现部门","现职级","现生效日"], "checkType": "not_null"} |
| cross_table | { "targetSheet": "目标表名", "joinField": "关联字段", "assertPresence": true } | {"targetSheet": "历史晋升信息", "joinField": "工号", "assertPresence": true} |

## 规则类型判断指南

- "不允许空值"、"非空"、"必填" → not_null
- "固定X位数字"、"格式为..." → regex
- "长度X到Y"、"不超过X个字符" → length_range
- "只能是..."、"取值范围为{A,B,C}" → enum_check
- "不可重复"、"唯一" → unique
- "日期格式为..." → date_format
- "数值在X到Y之间"、"范围" → value_range
- "当X时，Y必须有值"、"条件必填" → cross_field
- "应存在于XX表中"、"关联校验" → cross_table

## 注意事项
- 如果无法确定规则类型或参数，设 confidence 为 "low"
- 占位符请根据上下文推断，常见占位符包括："无"、"-"、"待补充"、"NA"、"null"、"N/A"、"/"
- severity 判断：必然错误 → error，疑似异常 → warning，合理性待确认 → info
`;
```

### 7.3.2 字段匹配 Prompt（field-match.ts）

**目的**：判断数据表列名与规则字段名之间的语义等价关系。

```typescript
// lib/ai/prompts/field-match.ts

export const FIELD_MATCH_SYSTEM_PROMPT = `你是数据字段匹配专家。你的任务是判断数据表的列名与校验规则的字段名之间是否存在语义等价关系。

## 输入
你将收到两组字段名列表：
1. 数据表列名（实际数据中的列）
2. 规则字段名（校验规则中定义的字段）

## 输出
输出 JSON 数组，每条匹配包含：

\`\`\`json
[
  {
    "dataColumn": "数据表列名",
    "ruleField": "规则字段名",
    "matchMethod": "ai_semantic",
    "confidence": 0.85,
    "reason": "语义说明"
  }
]
\`\`\`

## 匹配原则
- 只返回置信度 >= 0.7 的匹配
- 不要强行匹配不相关的字段
- 同一数据列只能匹配一个规则字段（一对多不成立）
- 同一规则字段只能被一个数据列匹配（多对一不成立）
- 置信度评分参考：
  - 0.95-1.0：几乎确定等价（如"员工编号"和"工号"）
  - 0.80-0.94：高度相似（如"部门名称"和"所在部门"）
  - 0.70-0.79：语义相近但需确认（如"生效日期"和"异动生效日"）

## 输出要求
- 仅返回匹配成功的记录
- 未匹配的字段不需要返回
- 不要添加任何解释性文字，只返回纯 JSON
`;
```

### 7.3.3 问题诊断 Prompt（issue-diagnosis.ts）

**目的**：对校验发现的问题进行通俗化根因分析和修改建议。

```typescript
// lib/ai/prompts/issue-diagnosis.ts

export const ISSUE_DIAGNOSIS_SYSTEM_PROMPT = `你是数据质量诊断专家。你的任务是对校验规则引擎发现的问题进行通俗化解读，并提供可操作的修改建议。

## 措辞规范（严格遵守）
- ✅ 使用："建议核查"、"可能存在异常"、"建议关注"、"待确认"、"建议补充"
- ❌ 禁止使用："违规"、"错误"、"整改"、"不合规"、"处罚"、"异常数据"（"疑似异常"可用）

## 分析维度
1. **根因分析（diagnosis）**：分析问题可能的成因
   - 录入环节遗漏（人工填写时漏填）
   - 系统同步问题（上下游系统数据不一致）
   - 历史脏数据（旧数据未清理）
   - 规则理解偏差（对业务标准理解有误）
   - 流程设计缺陷（流程未强制校验）

2. **修改建议（suggestion）**：提供具体的修复方案
   - 建议应具体到操作层面
   - 区分短期修复（修正当前数据）和长期改进（优化流程）

3. **问题分类（category）**：对问题进行归类
   - 录入类 / 同步类 / 历史数据类 / 规则类 / 流程类

## 输出格式
\`\`\`json
{
  "diagnosis": "根因分析文字",
  "suggestion": "修改建议文字",
  "category": "录入类|同步类|历史数据类|规则类|流程类"
}
\`\`\`

## 输入信息
你将收到以下信息：
- 表名（sheetName）
- 字段名（fieldName）
- 原始值（originalValue）
- 违规规则描述（ruleMessage）
- 问题等级（severity）

## 注意事项
- 诊断要客观温和，不做出绝对性判断
- 建议要具体可操作，避免空泛
- 同一类问题保持诊断风格一致
`;
```

## 7.4 AI 降级策略

当 AI 服务不可用时（API Key 无效、网络故障、模型限流），系统自动降级，确保核心校验流程不受影响。

### 7.4.1 降级实现

```typescript
// lib/ai/fallback.ts

import { logAuditEvent } from '../db/audit';

export enum AIFallbackReason {
  API_KEY_INVALID = 'api_key_invalid',
  NETWORK_ERROR = 'network_error',
  RATE_LIMITED = 'rate_limited',
  TIMEOUT = 'timeout',
  UNKNOWN = 'unknown',
}

export interface FallbackContext {
  scenario: 'parseStandard' | 'matchFields' | 'diagnoseIssue' | 'generateReport';
  error: Error;
  reason: AIFallbackReason;
}

export function handleFallback(ctx: FallbackContext): any {
  // 记录降级事件
  logAuditEvent('ai_fallback', {
    scenario: ctx.scenario,
    reason: ctx.reason,
    errorMessage: ctx.error.message,
    timestamp: new Date().toISOString(),
  });

  switch (ctx.scenario) {
    case 'parseStandard':
      return {
        rules: [],
        message: 'AI 标准解析服务不可用，请手动配置规则或使用规则模板',
        fallback: true,
      };

    case 'matchFields':
      return {
        matches: [],  // 空匹配，后续由精确匹配和别名匹配填充
        message: 'AI 语义匹配不可用，仅执行精确匹配和别名匹配',
        fallback: true,
      };

    case 'diagnoseIssue':
      return {
        diagnosis: 'AI 诊断服务不可用，请参考规则引擎的错误描述',
        suggestion: '',
        category: 'unknown',
        fallback: true,
      };

    case 'generateReport':
      return {
        report: generateStatReport(),  // 纯统计报告
        fallback: true,
      };
  }
}

/**
 * 生成纯统计报告（不含 AI 分析）
 */
function generateStatReport(): string {
  return `# 数据质量校验报告

> 注：AI 辅助分析服务不可用，本报告仅包含统计数据。

## 校验概况
- 数据表：{tableCount} 张
- 总记录数：{totalRecords} 条
- 校验规则：{ruleCount} 条
- 发现问题：{issueCount} 项

## 维度通过率
（详见问题汇总表）

## 建议
请修复 AI 服务配置后重新生成完整分析报告。`;
}
```

```
AI 调用
    ↓
超时/网络错误/限流/API Key无效
    ↓
handleFallback()
    ├── 记录降级事件到 audit_logs
    ├── 根据场景返回降级结果
    └── 前端展示降级提示
```

export async function checkAIHealth(config: AiConfig): Promise<{ healthy: boolean; message: string }> {
  try {
    const client = new OpenAI({
      apiKey: config.apiKey,

    }
    return { healthy: false, message: `连接失败：${message}` };
  }
}

## 7.5 AI 调用优化

### 7.5.1 批量调用

字段匹配时一次性发送所有列名对，而非逐条调用，减少请求次数。

```typescript
// 错误做法：逐条匹配
for (const col of assetColumns) {
  for (const field of ruleFields) {
    await dashScopeService.matchFields([col], [field]); // N*M 次调用！
  }
}

// 正确做法：批量匹配
const matches = await dashScopeService.matchFields(assetColumns, ruleFields, template);
// 1 次调用，返回所有 >= 0.7 置信度的匹配
```

### 7.5.2 结果缓存

相同输入缓存 AI 响应，避免重复调用。

```typescript
// lib/ai/cache.ts

import crypto from 'crypto';
import fs from 'fs/promises';
import path from 'path';

const CACHE_DIR = path.join(process.cwd(), '.ai-cache');
const CACHE_TTL = 7 * 24 * 60 * 60 * 1000; // 7 天

interface CacheEntry {
  key: string;
  response: any;
  timestamp: number;
}

function computeCacheKey(input: string): string {
  return crypto.createHash('md5').update(input).digest('hex');
}

export async function getCachedResponse(key: string): Promise<any | null> {
  try {
    const cachePath = path.join(CACHE_DIR, `${key}.json`);
    const content = await fs.readFile(cachePath, 'utf-8');
    const entry: CacheEntry = JSON.parse(content);

    // 检查过期
    if (Date.now() - entry.timestamp > CACHE_TTL) {
      await fs.unlink(cachePath);
      return null;
    }

    return entry.response;
  } catch {
    return null;
  }
}

export async function setCachedResponse(key: string, response: any): Promise<void> {
  await fs.mkdir(CACHE_DIR, { recursive: true });
  const cachePath = path.join(CACHE_DIR, `${key}.json`);
  const entry: CacheEntry = { key, response, timestamp: Date.now() };
  await fs.writeFile(cachePath, JSON.stringify(entry, null, 2));
}

export async function clearExpiredCache(): Promise<void> {
  const files = await fs.readdir(CACHE_DIR);
  for (const file of files) {
    const filePath = path.join(CACHE_DIR, file);
    const stat = await fs.stat(filePath);
    if (Date.now() - stat.mtimeMs > CACHE_TTL) {
      await fs.unlink(filePath);
    }
  }
}
```

缓存使用示例：

```typescript
async function matchFieldsWithCache(
  assetColumns: string[],
  ruleFields: string[]
): Promise<FieldMatch[]> {
  const input = JSON.stringify({ assetColumns, ruleFields });
  const cacheKey = computeCacheKey(input);

  const cached = await getCachedResponse(cacheKey);
  if (cached) return cached;

  const result = await dashScopeService.matchFields(
    assetColumns, ruleFields, FIELD_MATCH_SYSTEM_PROMPT
  );

  await setCachedResponse(cacheKey, result);
  return result;
}

### 7.5.3 并发控制

最多同时发起 2 个 AI 请求，避免触发限流。

```typescript
// lib/ai/concurrency.ts

class AIConcurrencyController {
  private activeRequests = 0;
  private readonly maxConcurrent = 2;
  private queue: Array<{
    fn: () => Promise<any>;
    resolve: (value: any) => void;
    reject: (reason: any) => void;
  }> = [];

  async execute<T>(fn: () => Promise<T>): Promise<T> {
    if (this.activeRequests < this.maxConcurrent) {
      this.activeRequests++;
      try {
        return await fn();
      } finally {
        this.activeRequests--;
        this.processQueue();
      }
    }

    // 排队等待
    return new Promise<T>((resolve, reject) => {
      this.queue.push({ fn, resolve, reject });
    });
  }

  private processQueue() {
    if (this.queue.length === 0 || this.activeRequests >= this.maxConcurrent) return;

    const { fn, resolve, reject } = this.queue.shift()!;
    this.activeRequests++;

    fn()
      .then(resolve)
      .catch(reject)
      .finally(() => {
        this.activeRequests--;
        this.processQueue();
      });
  }
}

export const aiConcurrencyController = new AIConcurrencyController();
```

使用方式：

```typescript
// 并发控制下的 AI 调用
const result = await aiConcurrencyController.execute(() =>
  dashScopeService.parseStandard(content, template)
);
```

### 7.5.4 超时与重试

```typescript
// lib/ai/retry.ts

const AI_TIMEOUT_MS = 30_000; // 30 秒
const AI_MAX_RETRIES = 1;
const AI_RETRY_DELAY_MS = 2000;

export async function callAIWithRetry<T>(
  fn: () => Promise<T>,
  timeoutMs: number = AI_TIMEOUT_MS
): Promise<T> {
  let lastError: Error | null = null;

  for (let attempt = 0; attempt <= AI_MAX_RETRIES; attempt++) {
    try {
      const timeoutPromise = new Promise<never>((_, reject) => {
        setTimeout(() => reject(new Error('AI 调用超时')), timeoutMs);
      });

      return await Promise.race([fn(), timeoutPromise]);
    } catch (error: any) {
      lastError = error;
      if (error.message === 'AI 调用超时') {
        console.warn(`AI 调用超时（第 ${attempt + 1} 次）`);
      } else {
        console.warn(`AI 调用失败（第 ${attempt + 1} 次）：${error.message}`);
      }

      if (attempt < AI_MAX_RETRIES) {
        await new Promise(resolve =>
          setTimeout(resolve, AI_RETRY_DELAY_MS * (attempt + 1))
        );
      }
    }
  }

  throw lastError || new Error('AI 调用未知错误');
}
```

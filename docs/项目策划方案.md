# 数据质量校验工具项目策划方案 V1.0

**版本**：V1.0  

**编制日期**：2026年5月27日  

## 一、方案总览

### 1.1 项目定位

本工具为业务标准驱动、分层分级刚性校验、可溯源可复盘、AI智能辅助的数据质量校验平台，服务于数据治理部门内部日常工作。工具核心基于业务方已定稿的数据质量标准开展落地校验工作，解决传统数据校验纯手工标记、标准落地断层、大文件上传易中断、校验维度混乱、问题定位模糊、成果无法量化的痛点。

工具采用规则引擎做主校验、AI大模型做辅助增效的双引擎架构，核心校验判定100%刚性规则可控可审计，同时通过AI解决标准翻译难、问题看不懂、整改效率低的落地难题，形成业务定标准、系统强校验、AI辅分析、全程可追溯的全流程数据质量治理体系。

工具面向业务方的措辞严格遵循辅助原则，使用"建议核查""可能存在异常""待确认"等温和表述，禁用"违规""整改""不合规"等绝对性词语，定位为帮业务方找出数据问题、提供修改建议的辅助角色。

### 1.2 核心建设目标

- **标准基准唯一化**：完全依托业务定稿标准作为唯一校验基准，完成标准归档、版本锁定，保障校验结果贴合业务、权威可信。
- **校验分层精细化**：严格拆分字段级、记录级、跨数据集级三层独立校验，解耦执行逻辑，精准定位各类数据问题。
- **素材资产常态化**：构建统一数据素材池，支持大文件分块异步上传、断点续传、复用，实现多版本数据集沉淀。
- **任务执行高可用**：基于多线程、分批次、异步调度机制，保障海量数据校验任务连续稳定执行，杜绝任务中断、结果错乱。
- **成果管控体系化**：输出标准化校验成果、量化质量统计、智能分析报告， 且所有输入输出固化保存，版本管理，可导出、可查询、可对比。支持一键导出、历史溯源、操作留痕。
- **AI增效智能化**：通过AI完成标准翻译、问题诊断、通俗化解读、修改建议生成，大幅降低人工治理成本，不干预核心校验判定。

### 1.3 核心输入输出定义

**输入**：

| 输入项 | 说明 |
|--------|------|
| 业务域选择 | 手动创建选择业务域（HR、财务、销售等），先选域再上传对应标准 |
| 数据质量标准 | 三级数据质量标准（字段级、记录级、跨数据集级），支持Excel等格式 |
| 待校验数据集 | 支持多表上传，一次校验可覆盖多个关联数据表 |

**输出**：

| 输出项 | 说明 |
|--------|------|
| 校验标记数据表 | 字段级问题单元格颜色标记、记录级问题行标记、跨数据集问题独立清单 |
| 数据质量问题汇总表 | 按表、字段、规则、问题类型统计数量、占比、合格率 |
| 数据治理分析报告 | 校验前质量评估、校验后结果分析、识别准确率、AI智能修改建议（温和措辞、可直接操作） |

## 二、整体架构设计

### 2.1 整体六层架构

1. **业务基准接入层**：承接业务定稿数据质量标准、待校验数据集素材，提供标准导入归档、版本锁定、标准-规则映射能力，确立唯一校验依据。

2. **素材资产与合规层**：搭建统一数据素材池，实现素材按业务域分类、版本管理、大文件断点续传、精细化权限管控，兼顾资产沉淀与数据合规。

3. **AI智能辅助层**：独立辅助能力层，负责自然语言标准翻译、问题诊断、通俗化解读、智能修改建议生成、分析报告输出，**不参与核心校验判定**。

4. **规则引擎核心层**：解析固化后的可执行校验规则，分层拆解三级校验逻辑，提供精准、可复现、可审计的校验规则支撑。

5. **分层校验执行层**：字段级、记录级、跨数据集级三层校验独立队列、分批次异步执行，配套批次容错机制，保障任务连续性。

6. **成果溯源与管理层**：实现问题精准标识、量化统计、成果一键导出、全链路版本溯源、操作审计留痕。

### 2.2 核心架构亮点

- **双引擎严格隔离**：规则引擎管控所有校验判定、统计核心逻辑；AI仅承担辅助增效工作（提效、易懂、降成本），杜绝AI幻觉导致的治理风险。
- **全链路版本绑定**：标准版本、素材版本、任务版本、成果版本四元绑定，所有校验结果可复现、可对比、可溯源。
- **全场景容错兜底**：覆盖脏数据兼容、任务重试防错乱，适配真实复杂业务场景。
- **三级问题标识**：字段级单元格标记、记录级行标记、跨数据集独立清单，问题定位精准到粒度。

## 三、核心功能模块详细设计

### 3.1 业务标准与规则映射管理模块

解决业务自然语言标准与系统可执行规则映射断层的问题，为全量校验提供唯一、可信、可落地的刚性基准。

#### 3.1.1 标准导入与版本管控

- 支持导入业务已定稿的三级数据质量标准（字段级、记录级、跨数据集级），Excel格式，系统自动解析入库；
- 标准按业务域归档，选择业务域后上传对应标准，系统自动建立标准台账；
- 标准确认后锁定版本号，所有校验任务必须绑定指定标准版本，杜绝基准混用；
- 同名文件重复上传自动追加版本号（v1, v2, v3），历史版本可回溯、可对比；
- 支持标准在线预览、修订迭代，全程可审计。

#### 3.1.2 标准-可执行规则映射

针对业务标准多为自然语言、无法直接代码执行的痛点：

- **AI辅助翻译**：上传业务标准文档后，AI自动解析自然语言规则，生成可执行校验逻辑初稿（正则、值域范围、逻辑公式、跨表关联条件），人工审核确认后固化生效；
- **逐条绑定**：每一条业务标准关联一条或多条系统可执行规则，映射关系全量留痕；
- **规则编辑**：AI解析的规则可在线编辑修正，修正后重新固化；已关联的校验任务提示是否需要用新规则重新校验；
- **规则复用**：固化后的规则保存到规则库，同一标准可反复校验不同批次数据，无需重新解析；
- **规则冲突检测**：同字段多规则冲突时自动告警，人工确认后执行；
- **规则增量更新**：标准更新时支持增量追加（新增规则、修改规则、停用规则），无需全量重新解析；已固化规则不受影响，新增规则走AI解析+人工确认流程后追加到规则库；
- **AI解析置信度与审核机制**：AI翻译规则时自动标注置信度（高/中/低），低置信度规则优先展示供人工审核，高置信度规则仍需人工确认但可批量快速通过，避免逐条看1000条规则审核不过来的问题。

### 3.2 数据素材池管理模块

针对多版本业务数据集和大文件场景，实现统一资产管理、高效传输、合规存储。

#### 3.2.1 素材资产管理

- 按业务域分类归档，建立素材资产台账；
- 同一数据集支持多版本存储，精准绑定对应标准版本与校验任务；
- 所有已入库素材支持全局复用，无需重复上传；
- 素材元数据管理：名称、业务域、上传时间、文件大小。

#### 3.2.2 大文件高性能上传能力

- **分块切片**：大文件自动切分固定大小分片，降低单次传输压力；
- **异步多线程上传**：多分片并行传输，不阻塞前端操作；
- **断点续传**：记忆已上传分片，网络中断后可接续传输，无需重传全量文件；
- **秒传机制**：基于MD5校验识别重复素材，直接复用已有文件。

#### 3.2.3 素材合规存储

- 敏感数据加密存储，手机号、身份证等隐私数据入库加密；
- 校验逻辑基于原始数据精准执行，前端展示与成果导出自动脱敏，兼顾校验准确性与数据安全。

### 3.3 字段智能匹配模块

解决待校验数据表列名与标准表字段名不一致的问题，保障规则正确关联到数据列。

| 匹配策略 | 优先级 | 说明 |
|----------|--------|------|
| 精确匹配 | 1 | 列名与字段名完全一致 |
| 别名匹配 | 2 | 通过别名配置表匹配（如NAME=姓名） |
| AI语义匹配 | 3 | AI理解语义等价（如员工编号≈工号） |
| 人工确认 | 4 | 无法自动匹配时提示人工选择 |

输出匹配映射表，含置信度，人工可修正。**未匹配字段单独列出，严禁强行匹配。**

### 3.4 分层分批次校验执行模块（核心校验能力）

严格按照字段级→记录级→跨数据集级三层独立解耦校验，保障全场景数据质量覆盖。

#### 3.4.1 三级分层独立校验机制

三层校验独立队列、独立线程、独立结果存储，单层任务故障不影响整体，支持单独重跑、单独复盘。

- **字段级校验**（细粒度）：逐字段校验类型、长度、值域、非空、唯一、格式、枚举合规性，精准标识问题字段、原始值、违规规则。覆盖六性维度：完整性、准确性、有效性、唯一性。

- **记录级校验**（单条粒度）：校验单条数据内部多字段业务逻辑一致性，如"异动类型为晋升时，现字段组必须有值"、"同一工号在所有记录中姓名应保持一致"。覆盖一致性维度、层级约束。

- **跨数据集校验**（多源粒度）：校验多表之间数据关联一致性，如"花名册的工号应出现在历史晋升信息中"。覆盖跨表一致性。

**跨数据集关联关系定义**：跨数据集校验的前提是明确表间关联关系，系统提供两种方式：
- **AI自动推断**：AI根据规则文本中的跨表引用描述（如"应存在于XX表中"），自动推断关联关系并生成关联配置初稿，人工确认后生效；
- **人工配置**：支持手动指定关联表、关联主键、关联条件，适用于复杂关联场景。

#### 3.4.2 分批次容错执行

- 海量数据自动分块批次执行，单批次异常自动重试、单独标记，不影响全局任务连续性；
- 批次控制参数：字段级50条规则/批次、记录级1000条记录/批次、跨数据集级10条规则/批次；
- 每批次执行完保存checkpoint，支持中断恢复；
- 同一批次仅生成一次结果，重跑自动清空旧数据，杜绝结果叠加错乱。

#### 3.4.3 校验任务进度可观测

- 实时展示当前校验进度：已完成维度、当前执行维度、批次进度百分比；
- 展示各维度校验状态：等待中/执行中/已完成/异常中断；
- 异常中断时展示中断位置和原因，支持一键续跑。

### 3.5 AI智能辅助治理模块（增效核心能力）

严格遵循规则做主、AI辅助原则，AI不参与任何合规判定、统计打分，仅负责增效、解读、分析。

#### 3.5.1 为什么必须接入AI

**核心校验坚决不用AI，但辅助治理环节必须轻量AI介入。**

数据质量校验是"强规则、强确定性、可审计、可追责"场景，AI不可做主，但有一个核心卡点只有AI能解决：**业务标准到可执行规则的翻译**。

不同业务域的规则表差异巨大——列名可能不一样（"字段"/"项目"/"列名"），规则描述可能不一样，表格格式可能不一样。传统代码不可能适配所有表格，只有AI能做到语义理解：不管上传什么规则表，都能自动识别哪列是表名、哪列是字段、哪列是规则、规则是什么意思，并翻译成系统可执行的校验逻辑。

此外，传统工具只报"错误代码"，业务看不懂、不会改。AI能把技术语言翻译成业务语言，生成通俗的问题解释和可操作的修改建议，解决"技术看得懂、业务看不懂，治理无法闭环"的通病。

#### 3.5.2 核心校验为什么绝对不能用AI（红线底线）

核心校验场景有四条硬性要求，AI无法满足：

| 硬性要求 | 说明 |
|----------|------|
| 100%可复现 | 同样的数据+同样的标准，必须每次跑出一模一样的结果。AI具有随机性，无法用于正式校验 |
| 100%可审计 | 每一条报错必须明确依据哪一条业务标准、哪一条规则公式。AI判断属于黑盒，无法追责 |
| 零误判容忍 | 数据治理考核、质量打分、业务整改都是刚性结果，AI幻觉会直接导致业务不认、审计不通过 |
| 高吞吐稳定 | 大批量、分批次、断点续跑任务，AI推理速度、并发、成本远不如规则引擎 |

**AI严禁介入的禁区**：字段级合规校验、记录级业务逻辑校验、跨数据集关联校验、问题判定与统计打分、任务调度与批次执行。

#### 3.5.3 AI能力边界与优先级

**AI能做的事**：

| 优先级 | 能力 | 说明 |
|--------|------|------|
| P0 | 标准智能解析 | 自动解析任意格式规则表，理解表格结构，识别表名/字段/规则列，批量生成可执行规则初稿，人工确认后固化 |
| P0 | 字段智能匹配 | 理解语义等价的字段名（如"员工编号"≈"工号"），自动关联数据列与规则字段 |
| P0 | 问题诊断与修复建议 | 对每条问题数据生成通俗化解释、根因分析（录入问题/同步问题/历史脏数据）、可操作修改建议 |
| P1 | 问题智能分级聚类 | 自动聚合同类问题、智能定级（致命/严重/一般/提示）、生成整改优先级排序 |
| P1 | 分析报告自动生成 | 自动生成治理分析报告，组织统计数据为可读内容，含AI修改建议 |
| P2 | 语义级判断 | 测试数据识别、占位符识别（含标准未穷举的模式）、业务合理性提示 |
| P2 | 异常特征挖掘 | 聚类脏数据/特例数据特征，辅助治理人员迭代优化业务标准与校验规则 |

**AI不能做的事**：

| 禁止项 | 原因 |
|--------|------|
| 确定性校验执行 | 非空、格式、长度、枚举等规则，程序化执行100%可靠，AI反而可能出错 |
| 绝对判定 | 不说"违规""不合规"，拿不准的标为"待人工确认" |
| 代替业务决策 | 数据在某些业务场景下可能合理，AI不判断业务上是否可接受 |
| 篡改原始数据 | AI只标记、建议，绝不直接修改业务数据 |
| 编造规则 | 校验依据只能是上传的数据标准，AI不能自创规则执行 |
| 保证100%覆盖 | 语义判断有边界，必须承认漏检可能 |

#### 3.5.4 最终架构：规则驱动校验+AI辅助治理

- **规则引擎负责**（100%精准、可审计、可追责）：三级校验判定、问题标识统计打分、任务调度批次执行、所有正式成果输出；
- **AI大模型负责**（增效、辅助、翻译、诊断）：标准文档解析与规则翻译、问题根因分析与通俗解读、修复建议生成与优先级排序、问题聚类与异常挖掘、分析报告自动输出。

### 3.6 数据质量成果输出模块

覆盖问题标识、汇总统计、分析报告、修改建议、成果导出全能力。

#### 3.6.1 三级问题标识体系

| 级别 | 标记方式 | 标识内容 |
|------|----------|----------|
| 字段级 | 单元格颜色填充 | 数据表、行号、字段名、原始值、违规规则、问题等级 |
| 记录级 | 整行首列标记 | 唯一记录标识、违规逻辑、涉及字段 |
| 跨数据集级 | 独立问题清单 | 涉及数据集、关联主键、数据差异说明 |

**颜色规范**：

| 等级 | 颜色 | 含义 |
|------|------|------|
| 严重 | 红色 #FFC7CE | 必然错误（如空值、格式不符） |
| 警告 | 黄色 #FFEB9C | 疑似异常（如可能的占位符） |
| 提示 | 蓝色 #BDD7EE | 建议关注（如业务合理性待确认） |

#### 3.6.2 标准化成果清单

- **问题汇总统计表**：按表、字段、规则级别、六性维度统计问题数量、占比、合格率；
- **数据治理分析报告**：含校验前质量评估、校验后结果分析、各维度对比、Top问题分析、AI智能修改建议（温和措辞、可直接操作）；

#### 3.6.3 成果导出

- 所有成果支持一键导出（Excel/PDF）；
- 数据质量标准可导出；
- 校验成果物可导出；
- 全部输入输出固化保存到历史记录。

### 3.7 历史记录与版本管理模块

- **全局版本管理**：同名文件重复上传自动追加版本号（v1, v2, v3），不覆盖历史；
- **四元版本绑定**：标准版本+素材版本+任务版本+成果版本，任意历史成果可溯源、可复现、可对比；
- **记录类型**：上传记录、校验记录、操作记录，全程留痕；
- **查询筛选**：按业务域、时间范围、操作类型筛选；
- **导出能力**：历史记录可导出，满足审计需求。

### 3.8 权限与安全模块

| 措施 | 说明 |
|------|------|
| 业务域数据隔离 | 不同业务域数据不可互访 |
| 角色权限控制 | 管理员（全功能）、治理专员（执行校验+查看报告）、业务方（仅查看报告） |
| 传输加密 | HTTPS全链路 |
| 存储加密 | 敏感数据AES-256 |
| 审计日志 | 操作不可篡改 |

## 四、核心技术落地方案

### 4.1 大文件分块异步+断点续传方案

前端完成文件切片与MD5校验，后端通过多线程线程池异步接收分片，数据库持久化分片上传状态，中断后自动续传，全部分片上传完成后校验完整性并自动合并文件，最终入库素材池。

### 4.2 分层分批次异步校验方案

将整体校验任务拆分为三层独立子任务（字段级→记录级→跨数据集级），数据集按固定行数分块批量推送校验引擎；单批次异常独立重试、隔离故障；通过任务唯一ID实现幂等控制，重跑任务自动清空历史批次结果，保障任务持续稳定执行。

### 4.3 双引擎隔离运行方案

规则引擎独立部署运行，承载所有校验判定、统计核心刚性逻辑，结果可复现、可审计；AI引擎独立部署，仅承接解析、诊断、归纳、报告生成等辅助工作，双引擎互不干扰，兼顾系统稳定性与智能增效能力。AI不可用时自动降级到规则引擎，不卡死任务。

## 五、标准化业务流程

1. **选择业务域**：选择HR/财务/销售等业务域，系统加载对应标准集；
2. **标准导入归档**：上传数据质量标准，AI自动解析自然语言规则，人工审核确认后固化，完成版本锁定；
3. **素材入库**：上传待校验数据集（支持多表），大文件断点续传，沉淀至素材池；
4. **字段智能匹配**：自动匹配数据列与规则字段，输出映射表，人工可修正；
5. **分层异步校验**：分维度、分批次、异步执行字段级→记录级→跨数据集级全量校验；
6. **AI智能分析**：自动诊断问题根因、通俗化解读、生成修改建议；
7. **成果生成输出**：生成标记数据表、问题汇总表、分析报告（含AI修改建议）；
8. **导出归档溯源**：一键导出全套成果，系统留存全量历史记录，版本管理。

## 六、核心风险规避

| 风险 | 规避措施 |
|------|----------|
| 标准落地断层 | AI辅助翻译自然语言规则+人工审核确认，解决标准无法直接执行的问题 |
| 字段匹配错误 | 四级匹配策略+严禁强行匹配，未匹配字段单独列出待人工确认 |
| 大文件上传失败 | 分块切片+断点续传+多线程异步，网络中断可续传 |
| AI服务不可用 | 自动降级到规则引擎，跳过语义判断类规则并标记"待AI复核" |
| 任务结果错乱 | 幂等性机制，重跑自动清空旧结果，checkpoint断点恢复 |
| 校验结果不可信 | 双引擎隔离，核心判定100%规则引擎执行，可复现可审计 |
| 业务措辞争议 | 禁用词与建议词表管控输出措辞，定位辅助角色 |
| 敏感数据泄露 | 数据加密存储、展示脱敏、业务域隔离、权限管控 |
| AI解析规则出错 | AI翻译规则时标注置信度，低置信度优先人工审核；所有规则人工确认后才能固化生效 |

**触发**：用户上传数据质量标准Excel

### 9.4 模块3：字段智能匹配

**触发**：用户上传待校验数据后，系统自动匹配

|------|------|

### 9.5 模块4：分层校验执行

**触发**：字段匹配完成后，用户点击"开始校验"

| 项目 | 说明 |
|------|------|
| 输入 | 待校验数据（DataFrame） + 固化规则库 + 字段映射表 |
| 处理步骤 | ① 字段级校验（逐规则逐行执行）→ ② 记录级校验（逐条记录多字段联动）→ ③ 跨数据集级校验（多表关联）→ ④ 每步记录执行日志 |
| 输出 | 校验结果明细：`{规则ID, 表名, 行号, 字段名, 原始值, 校验结果(通过/不通过), 问题等级, 问题原因}` |
| 批次机制 | 字段级50条/批、记录级1000条/批、跨数据集10条/批；每批保存checkpoint |
| 进度展示 | 当前维度、批次进度百分比、各维度状态（等待/执行中/完成/异常） |
| 存储 | 校验结果表 + 执行日志表，绑定任务ID |

**校验逻辑分类**：

| 规则类型 | 可执行逻辑type | 示例 |
|----------|---------------|------|
| 非空 | not_null | 工号不允许空值 |
| 格式 | regex | 工号固定8位纯数字 |
| 长度 | length_range | 名称2-15字符 |
| 枚举 | enum_check | 异动类型限定枚举值 |
| 唯一 | unique | 工号不可重复 |
| 日期格式 | date_format | 日期必须YYYY-MM-DD |
| 值域 | value_range | 年龄0-150 |
| 跨字段逻辑 | cross_field | 异动类型为晋升时现字段组必须有值 |
| 跨表关联 | cross_table | 花名册工号应出现在晋升信息中 |

### 9.6 模块5：结果标记与输出

**触发**：校验执行完成后自动生成

| 项目 | 说明 |
|------|------|
| 输入 | 校验结果明细 + 原始数据表 |
| 处理 | ① 按规则ID+行号+字段定位问题单元格 → ② 按问题等级填充颜色 → ③ 统计各维度问题数量/占比/合格率 → ④ AI生成分析报告和修改建议 |
| 输出1 | 标记数据表（Excel，问题单元格颜色填充，含批注说明违规规则） |
| 输出2 | 问题汇总统计表（Excel，按表/字段/维度/等级统计） |
| 输出3 | 数据治理分析报告（Markdown，含校验前后评估、Top问题、AI修改建议） |
| 存储 | 成果表，绑定任务ID和版本号 |

**颜色标记规则**：
- 严重（红色#FFC7CE）：空值、格式不符、必填缺失
- 警告（黄色#FFEB9C）：疑似占位符、可能的异常值
- 提示（蓝色#BDD7EE）：业务合理性待确认

**修改建议措辞规范**：
- ✗ 禁用：违规、错误、整改、处罚、不合规
- ✓ 使用：建议核查、可能存在异常、建议关注、待确认、建议补充
